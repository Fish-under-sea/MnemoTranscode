# MTC — Memory To Code 详细规格说明书

> 人的记忆是一种不讲道理的存储介质。这个项目存在的意义，就是把这些失衡的记忆碎片提取出来，完成从生物硬盘到数字硬盘的格式转换。

---

## 一、项目概述

### 1.1 项目名称与定位

**MTC**（MnemoTranscode — Memory To Code）是一个通用的 AI 记忆传承平台。它不只服务于家族记忆，可以承载恋人、挚友、至亲、伟人乃至一个国家/民族的历史记忆。

> **核心愿景**：用 AI 技术将记忆（声音、照片、文字、情感）进行数字化存档、智能化整理和多模态还原，让这些瞬间被脱离情绪、平铺在显示屏上时，你或许会发现——留下的，只是一个具体的、真实的、曾在生命里留下过折痕的人。

### 1.2 支持的档案类型

| 档案类型 | 标识 | 说明 |
|----------|------|------|
| 家族记忆 | `family` | 家族世代传承的记忆 |
| 恋人记忆 | `lover` | 恋人之间的珍贵回忆 |
| 挚友记忆 | `friend` | 朋友之间的深厚情谊 |
| 至亲记忆 | `relative` | 直系亲属的故事 |
| 伟人记忆 | `celebrity` | 历史名人、伟人的生平 |
| 国家历史 | `nation` | 一个国家/民族的历史记忆 |

每种类型共享同一套数据模型和功能逻辑，差异化体现在呈现方式和 AI 对话的人设模板上。

## 二、功能规格

### 2.1 输入层功能

#### 2.1.1 语音对话录入

- 支持多轮对话引导提问
- 自动语音转文字（STT）
- 说话人分离（区分不同家庭成员）
- 语音质量增强

#### 2.1.2 照片与视频

- 支持批量上传照片/视频素材
- 自动提取 EXIF 信息（时间、地点）
- AI 图像描述生成
- 视频帧提取与标注

#### 2.1.3 文字录入

- 手写文字识别
- 键盘直接录入
- 支持 Markdown 格式
- 模板化录入（年谱式、叙事式、书信式）

#### 2.1.4 声音采集

- 声纹建模采集（需授权确认）
- 多人声纹分离
- 音频质量自动检测

### 2.2 AI 核心层功能

#### 2.2.1 记忆整理引擎

- LLM 自动总结记忆内容
- 关键词提取
- 时间线索推断
- 人物关系识别
- 地点归类

#### 2.2.2 情感识别模块

- 文本情感分析（10 种情绪标签）
- 情感强度检测
- 情绪变化趋势分析
- 多人对话中的情感追踪

#### 2.2.3 声音克隆模块

- TTS 文字转语音
- 声纹迁移（用少量样本克隆声音）
- 情感语音合成
- 多语言支持

### 2.3 存储层功能

#### 2.3.1 向量记忆库

- 记忆语义向量化存储
- 语义相似度检索
- 时间戳索引
- 多维度检索过滤

#### 2.3.2 多媒体素材库

- 照片/音频/视频的原始文件存储
- 元数据管理
- CDN 加速分发
- 缩略图生成

#### 2.3.3 加密家族档案

- 端到端加密存储
- 私钥授权访问控制
- 家族成员权限分级（所有者、编辑者、查看者）
- 数据导出与备份

### 2.4 输出层功能

#### 2.4.1 家族故事书

- LLM 自动生成故事文本
- 多种风格可选（怀旧温情/文学/简洁/对话）
- 自动排版生成 PDF
- 支持插入照片

#### 2.4.2 AI 亲人对话

- 用克隆的声音 + 记忆还原与"逝者"对话
- 支持多渠道接入（应用内/微信/QQ）
- 对话上下文记忆
- 伦理边界提示

#### 2.4.3 记忆时间线

- 可视化交互展示
- 按成员/年份/情感筛选
- 点击展开记忆详情
- 时间线自动重建

#### 2.4.4 记忆胶囊

- 设定未来解封时间
- 加密存储
- 定时推送（邮件/微信/应用内）
- 跨代传承设计

## 三、多渠道接入规格

### 3.1 原生软件对话

- Web 应用内直接对话
- React 前端 + FastAPI 后端
- 实时流式响应
- 语音播报（可选）

### 3.2 微信聊天转接

- 基于 KouriChat 项目技术
- wxautox4 库实现微信消息监听和发送
- 私聊和群聊双模式
- @触发 / 关键词触发 / 全自动模式
- 与 MTC 后端 API 对接

### 3.3 QQ 聊天转接（待开发）

- 基于 NoneBot 或 go-cqhttp
- 群聊接入
- 与 MTC 后端统一 API

## 五、技术规格

### 5.1 技术栈

| 层级 | 技术选型 |
|------|----------|
| 前端框架 | React 18 + Vite + TypeScript |
| UI 组件库 | Tailwind CSS + shadcn/ui |
| 状态管理 | Zustand + TanStack Query |
| 后端框架 | FastAPI + Python 3.11 |
| 数据库 | PostgreSQL 16 |
| 向量数据库 | Qdrant |
| 对象存储 | MinIO / S3 兼容 |
| 缓存/队列 | Redis + Celery |
| 容器化 | Docker + Docker Compose |
| CI/CD | GitHub Actions |

### 5.2 AI 模型集成

| 能力 | 推荐模型 | 说明 |
|------|----------|------|
| 文本生成 | GPT-4o / Qwen3 | LLM 对话和内容生成 |
| 向量嵌入 | text-embedding-3-small / BGE | 中文语义向量化 |
| 语音识别 | Whisper / SenseVoice | 语音转文字 |
| 语音合成 | CosyVoice / ElevenLabs | 文字转语音 |
| 情感分析 | deepctrl / GLM4 | NLP 情绪标注 |

### 5.3 API 规范

- RESTful API 设计
- 基于 JWT 的 Token 认证
- Bearer Token 模式
- 统一的错误响应格式
- 分页：基于 offset 和 limit
- 详细见 `docs/API.md`

## 六、非功能需求

### 6.1 性能

- API 响应时间 < 500ms（P95）
- 向量检索 < 100ms（P95）
- 前端首屏加载 < 3s
- 支持 1000+ 记忆条目的检索

### 6.2 安全

- 所有 API 需要认证
- 密码 bcrypt 哈希存储
- JWT 令牌 24 小时过期
- 敏感数据加密存储
- CORS 白名单控制

### 6.3 可用性

- Docker 一键部署
- 数据库自动迁移
- 健康检查端点
- 优雅停机

### 6.4 可扩展性

- 微服务架构预留
- 向量数据库横向扩展
- 媒体 CDN 加速
- 多租户支持预留

## 七、伦理与合规

### 7.1 声音克隆授权

- 声音采集必须获得明确授权
- 生成内容需加水印标识
- 服务条款明确禁止滥用
- 提供声音克隆内容举报机制

### 7.2 数据隐私

- 用户数据最小化采集
- 端到端加密（可选）
- 支持数据导出和删除
- 符合 GDPR 和中国网络安全法

### 7.3 AI 对话伦理边界

- 明确告知用户对话对象为 AI
- 禁止生成有害内容
- 提供对话记录存档
- 家属知情同意机制
